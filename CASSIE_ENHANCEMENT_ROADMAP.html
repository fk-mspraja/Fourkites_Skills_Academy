Executive Summary

Current Achievement: Cassie represents a sophisticated, production-grade RCA investigation platform actively used by support teams. The platform successfully integrates ReAct-based agents, multi-layered architecture, and enterprise data access through MCP servers.

Enhancement Opportunity: This document proposes targeted improvements to increase diagnostic accuracy and auto-resolution rates from current levels to 60-70% through systematic knowledge capture and pattern-based investigation enhancement.

Expected Impact: Enhanced case resolution efficiency, reduced manual investigation time, improved consistency across agents, and systematic capture of institutional diagnostic knowledge.

Part 2: Enhancement Recommendations

2.1 Strategic Enhancement: Pattern-Based Investigation Intelligence

Core Concept: Augment existing specialized RCA agents with structured diagnostic intelligence derived from historical case analysis and expert knowledge.

Enhancement Approach:

Recommendation 1: Diagnostic Pattern Library

What: Create structured repository of known diagnostic patterns with proven investigation protocols

Content Examples:

  • ELD Not Enabled Pattern: Specific query to check network configuration, clear evidence criteria, resolution template
  • Asset Assignment Pattern: Systematic verification of truck/trailer/device assignment, escalation logic
  • GPS Provider Error Pattern: Log analysis protocol, common error signatures, provider-specific guidance
  • Outlier Detection Pattern: Speed-based rejection checks, timestamp validation, data quality assessment

Implementation Approach:

  1. Extract patterns from Arpit's category sheet (100+ documented patterns)
  2. Enhance each pattern with: specific diagnostic queries, evidence evaluation criteria, confidence scoring logic, resolution templates
  3. Categorize by feasibility: HIGH (auto-resolve candidates), MEDIUM (guidance providers), LOW (engineering escalation)
  4. Embed relevant patterns into specialized agent prompts as investigation protocols

Expected Impact: 40-50% of cases match known patterns and can follow systematic investigation protocols

Recommendation 2: Evidence-Based Confidence Scoring

What: Implement systematic confidence evaluation for investigation findings

Confidence Framework:

  • HIGH Confidence (90-100%): Boolean checks with definitive results (e.g., ELD disabled, asset not assigned)
  • MEDIUM Confidence (70-89%): Pattern matches with supporting evidence from multiple sources
  • LOW Confidence (<70%): Ambiguous findings, conflicting evidence, requires human judgment

Decision Logic:

  • HIGH confidence → Auto-resolution candidate (draft response for human approval)
  • MEDIUM confidence → Detailed diagnostic summary with recommended next steps
  • LOW confidence → Escalation with collected evidence and investigation notes

Implementation: Enhance agent prompts with confidence evaluation criteria for each diagnostic pattern

Expected Impact: Clear decision boundaries, reduced ambiguous escalations, systematic auto-resolution identification

Recommendation 3: Systematic MCP Query Protocols

What: Provide agents with specific query patterns for common diagnostic scenarios

Query Protocol Examples:

ELD Configuration Check (via Redshift MCP):

  • Query Target: network_configurations table
  • Key Fields: eld_tracking_enabled, shipper_id, carrier_id
  • Evidence Evaluation: If eld_tracking_enabled = FALSE → Root cause identified (Confidence: 100%)
  • Resolution Path: Load ELD enablement template

Asset Assignment Verification (via Redshift MCP):

  • Query Target: load_tracking table
  • Key Fields: truck_number, trailer_number, device_id
  • Evidence Evaluation: If all NULL → Asset not assigned (Confidence: 100%)
  • Resolution Path: Load asset assignment template

GPS Provider Log Analysis (via Redshift MCP):

  • Query Target: gps_provider_api_logs table
  • Key Fields: error_message, timestamp, provider_name
  • Evidence Evaluation: Error signature matching against known patterns
  • Resolution Path: Provider-specific guidance templates

Implementation: Embed query protocols in specialized agent prompts, organized by investigation domain

Expected Impact: Systematic data collection, consistent evidence quality, reduced investigation time

Recommendation 4: Response Template Library

What: Pre-validated response templates for high-confidence pattern matches

Template Categories:

  • Configuration Issues: ELD enablement, network settings, feature flags
  • Assignment Issues: Asset assignment, carrier capability verification
  • Data Quality Issues: GPS provider guidance, log analysis summaries
  • Integration Issues: API connectivity, authentication validation

Template Structure:

  • Clear problem description in customer-friendly language
  • Root cause explanation with supporting evidence
  • Step-by-step resolution instructions
  • Expected timeline and verification steps
  • Escalation path if resolution doesn't work

Usage Model: When confidence ≥90%, agent drafts response using appropriate template with case-specific details populated

Expected Impact: Consistent response quality, reduced drafting time, improved customer communication
