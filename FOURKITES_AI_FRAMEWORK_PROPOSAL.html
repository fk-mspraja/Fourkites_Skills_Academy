04 / 32

Random Brains, Same Data Sources

The Pattern

Five analysts investigating "load not tracking":

  • All query the same data (Tracking API, Athena, Redshift)
  • Each uses DIFFERENT investigation patterns
  • Each has unique mental models and shortcuts
  • Some check ELD first, others start with callbacks
  • Experience determines investigation order

The Problem

  • No two investigations are identical
  • Senior analysts are 3x faster (but we can't clone them)
  • Knowledge is tribal, not systematic
  • New analysts take 6-12 months to ramp up
  • AI agents can't learn from unstructured expertise
  • Each team rebuilds the same wheels
The Opportunity:

If we can capture HOW the best analysts think (not just WHAT data they query), we can encode that into reusable Skills that both humans AND AI agents can leverage.

MCP (Model Context Protocol)

Anthropic's standard for connecting AI models to data sources:

  • AI agents call MCP servers like APIs
  • Servers translate requests to data source queries
  • Results returned in AI-friendly format
  • Handles authentication, rate limiting, context

Why This Matters

  • AI agents can't query raw databases effectively
  • Need structured, contextual interfaces
  • MCP servers provide domain-specific logic
  • One MCP server ‚Üí All AI agents benefit
  • Centralized access control and monitoring
Status:

We already have 10 MCP servers built (Athena, Redshift, Tracking API, etc.). Layer 4 is 70% complete.

Example 2: Callback/Webhook Failures

Real Production Issue - Jan 22, 2026

The Situation

Customer reports: "Callbacks not sent for 3 loads"

  • Tracking IDs: 626717801, 623705749, 625342806
  • Different error patterns: DNS failure, event not subscribed, HTTP 503

Manual Investigation

  • Analyst logs into Athena
  • Writes custom query for callbacks_v2 table
  • Analyzes error messages manually
  • Searches Confluence for similar issues
  • Determines if customer-side or FourKites-side
  • Drafts response email
  • Time: 25 minutes

Automated with Framework

  • Agent loads callback_failure playbook
  • Queries Athena via MCP server
  • Pattern matching: webhook_5xx (confidence: 0.90)
  • Retrieves resolution template from Skills Library
  • Generates customer communication automatically
  • Identifies: Customer endpoint issue (HTTP 503)
  • Time: 8 minutes
Key Insight:

After this investigation, we created 3 new pattern files (dns_lookup_failure, event_not_subscribed, http_503_error). Next time these occur, instant diagnosis with 95% confidence.
