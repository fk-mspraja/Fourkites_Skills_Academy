Critical Gaps to Address

GAP No GPS Position Database Access

Cannot query tracking_positions table to validate GPS timestamps or check location freshness.

SOLUTION: Build device-diagnostics-mcp with get_gps_position_history tool

GAP No Device Type Registry

Cannot determine if carrier supports truck vs trailer GPS, causing mismatch failures.

SOLUTION: Add get_device_type tool to query carrier_tracking_config

GAP No Network ELD Config Visibility

Cannot check if ELD tracking is enabled at the shipper-carrier relationship level.

SOLUTION: Add check_eld_enablement tool to query company_relationships

GAP No Unified Trace ID

Scattered identifiers (tracking_id, load_number, container) prevent E2E visibility.

SOLUTION: Correlation ID propagation via orchestrator state management

GAP Cassie Skips LOAD_NOT_TRACKING

Explicit code at main.py:1926 routes to manual intervention without investigation.

SOLUTION: Remove from skip list OR route to new multi-agent system

GAP Multiple Log Databases

30-day (SigNoz), 90-day (ClickHouse), 180-day (Athena) require different query strategies.

SOLUTION: Rewind MCP already handles smart fallback; extend pattern
