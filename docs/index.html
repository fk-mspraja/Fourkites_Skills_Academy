Cassie Agent Architecture

Complete Five-Layer Architecture with Skills Library Framework

Architecture Overview

Key Innovation: The Skills Library serves as a central, reusable knowledge repository that captures institutional expertise and enables rapid AI agent development. This layered architecture separates concerns: classification, knowledge, execution, intelligence, and decision-making operate independently.

Layer Legend

  • Layer 1: Classification & Routing - Intent classification and domain selection
  • Layer 2: Skills Library - Central knowledge repository (100+ patterns)
  • Layer 3: Investigation Execution - Playbook execution and MCP queries
  • Layer 4: Intelligence & Pattern Matching - Evidence evaluation and pattern matching
  • Layer 5: Decision & Response - Confidence-based auto-resolution or escalation

What is the Skills Library?

The Skills Library is Layer 2 in the 5-layer architecture - a centralized, reusable knowledge repository containing:

  • 100+ Diagnostic Patterns - ELD Not Enabled, Asset Not Assigned, GPS Provider Errors, Outlier Detection, etc.
  • Investigation Playbooks - Domain-specific protocols with step-by-step diagnostic checks
  • Response Templates - Auto-resolution text for common patterns
  • Evidence Rules - Confidence scoring and auto-resolve criteria (90%+ threshold)
  • Bot Feasibility Ratings - HIGH/MEDIUM/LOW automation potential for each pattern

How is the Skills Library Used?

The Skills Library is accessed 3 times during investigation:

  1. Load Playbook: After classification, Category Router queries Skills Library to select appropriate diagnostic playbook
  2. Pattern Matching: After collecting evidence, Pattern Matcher queries Skills Library to identify known patterns
  3. Response Generation: After high-confidence match, Response Generator loads template from Skills Library

Strategic Benefits

  • Reusability: Same skills used across multiple AI agents - write once, use everywhere
  • Maintainability: Update one pattern in Skills Library, all consumers benefit immediately
  • Rapid Development: New agents leverage existing diagnostic intelligence (weeks → days)
  • Knowledge Capture: Institutional expertise preserved and accessible systematically
  • Continuous Improvement: Patterns refined based on operational data, all agents improve simultaneously

Skills Library Benefits

  • Reusability: Same skills used across multiple AI agents and investigation domains
  • Maintainability: Update one pattern in Skills Library, all consumers benefit immediately
  • Rapid Development: New agents leverage existing diagnostic intelligence without rebuilding from scratch
  • Knowledge Capture: Institutional expertise preserved systematically and accessible to all agents
  • Continuous Improvement: Patterns refined based on operational data, benefiting all investigations
  • Separation of Concerns: Knowledge layer independent of execution, classification, and data layers

Implementation Impact

Current State: 0% auto-resolution, 100% manual escalation

With Skills Library:

  • Phase 1 (Weeks 1-2): 40-50% auto-resolution (basic diagnostic checks)
  • Phase 2 (Weeks 3-5): 80% diagnostic guidance (playbook execution)
  • Phase 3 (Weeks 6-8): 60-70% auto-resolution (full pattern library)
  • Phase 4 (Weeks 9-11): Production deployment with monitoring
