Cassie Agent Architecture

Complete Five-Layer Architecture with Skills Library Framework

Architecture Overview

Key Innovation: The Skills Library serves as a central, reusable knowledge repository that captures institutional expertise and enables rapid AI agent development. This layered architecture separates concerns: classification, knowledge, execution, intelligence, and decision-making operate independently.

Five-Layer Architecture

  • Layer 1: Classification & Routing - Intent classification and domain selection
  • Layer 2: Skills Library - Central knowledge repository (100+ patterns)
  • Layer 3: Investigation Execution - Playbook execution and MCP queries
  • Layer 4: Intelligence & Pattern Matching - Evidence evaluation and pattern matching
  • Layer 5: Decision & Response - Confidence-based auto-resolution or escalation

What is the Skills Library?

The Skills Library is Layer 2 in the 5-layer architecture - a centralized, reusable knowledge repository containing:

  • 100+ Diagnostic Patterns - ELD Not Enabled, Asset Not Assigned, GPS Provider Errors, Outlier Detection, etc.
  • Investigation Playbooks - Domain-specific protocols with step-by-step diagnostic checks
  • Response Templates - Auto-resolution text for common patterns
  • Evidence Rules - Confidence scoring and auto-resolve criteria
  • Bot Feasibility Ratings - HIGH/MEDIUM/LOW automation potential

How Skills Library is Used

The Skills Library is accessed at three critical points during investigation:

  1. Load Playbook: After classification, Category Router queries Skills Library to select appropriate diagnostic playbook
  2. Pattern Matching: After collecting evidence, Pattern Matcher queries Skills Library to identify known patterns
  3. Response Generation: After high-confidence match, Response Generator loads template from Skills Library

Strategic Benefits

  • Reusability: Write once, use across multiple AI agents
  • Maintainability: Update one pattern, all agents benefit immediately
  • Rapid Development: New agents leverage existing diagnostic intelligence
  • Knowledge Capture: Institutional expertise preserved systematically
  • Continuous Improvement: Patterns refined based on operational data

Layer Responsibilities

Layer 1 - Classification & Routing:

  • Intent classification using Cassie Agent
  • Domain selection via Category Router
  • Company identification and context gathering

Layer 2 - Skills Library:

  • Central repository of 100+ diagnostic patterns
  • Investigation playbooks for each domain
  • Response templates and evidence rules
  • Bot feasibility ratings

Layer 3 - Investigation Execution:

  • Playbook-driven diagnostic checks
  • MCP server queries (Redshift, Salesforce, Knowledge, Support AI, Atlassian)
  • Systematic data collection

Layer 4 - Intelligence & Pattern Matching:

  • Evidence aggregation from multiple sources
  • Pattern matching against Skills Library
  • Confidence scoring based on evidence quality

Layer 5 - Decision & Response:

  • Confidence-based decision logic
  • Auto-response generation for high-confidence matches
  • Manual review escalation for ambiguous cases
