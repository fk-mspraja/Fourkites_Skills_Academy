Knowledge Extraction Process

11 / 24

5-Step Systematic Process

Step 1

Knowledge Extraction (2-4 days)

Shadow SME sessions using 12-section template. Capture mental models, decision trees, data sources, and diagnostic patterns systematically.

Step 2

Playbook Creation (1 day)

Convert investigation flow to operational playbook YAML. Define steps, outcomes, branching logic, and API/query templates.

Step 3

Pattern Creation (1-2 days)

Convert diagnostic rules to pattern YAML files. Define evidence rules (confirms/rejects), confidence weights, and remediation templates.

Step 4

Test Case Creation (0.5 days)

Create test cases from real production issues. Define expected outcomes and validation criteria.

Step 5

Validation (0.5 days)

Run test cases against playbooks/patterns. Target: 90%+ accuracy. Refine weights and conditions based on results.

Total Timeline: 5-8 days from zero knowledge to production-ready playbook

Flexibility by Design

18 / 24

Everything is YAML-Configurable

What Can Be Changed

  • Add/remove diagnostic patterns
  • Adjust confidence thresholds
  • Change investigation priority order
  • Update resolution steps
  • Modify evidence weights
  • Configure data sources

No Code Changes Required

  • Edit YAML files directly
  • Patterns guide agent behavior
  • Support teams can contribute
  • Continuous improvement loop
  • Version controlled in Git
  • Test before deployment
